# Click & Buy: Микросервисная игра-кликер

Веб-приложение с микросервисной архитектурой, реализованное на **C# (.NET 10)**. Проект представляет собой игру-кликер, где пользователи могут "накликивать" баланс и покупать предметы в магазине, которые увеличивают доход от каждого клика. Также предусмотрена панель администратора для управления пользователями и ключами API.

## Архитектура и технологии

Проект разделен на несколько независимых компонентов, работающих в изолированных Docker-контейнерах:

* **Бэкенд (ASP.NET Core Minimal API):**
  * `ClickService` — сервис приема кликов. Не работает с БД напрямую, а публикует события в очередь RabbitMQ.
  * `PlayerService` — сервис управления игроками, балансом, инвентарем и API-ключами. Содержит фоновый воркер, асинхронно читающий клики из RabbitMQ и обновляющий базу данных.
  * `StoreService` — сервис магазина предметов с кэшированием каталога.
  * `AdminService` — сервис управления системой (блокировка/удаление пользователей).
* **Фронтенд (Blazor WebAssembly + R3 Reactive):**
  * `Frontend` — пользовательский интерфейс игры (порт 8080).
  * `AdminFrontend` — интерфейс администратора (порт 8081).
* **Инфраструктура:**
  * **Nginx** — используется как реверс-прокси внутри фронтенд-контейнеров для маршрутизации запросов к бэкенду и скрытия внутренних портов.
  * **RabbitMQ** — брокер сообщений для асинхронной связи (порт управления 15672).
  * **PostgreSQL** — 2 отдельные базы данных (`StoreDb` для магазина, `PlayerDb` для игроков).

## Безопасность

* **JWT-токены:** Авторизация между фронтендом и бэкендом (сохраняются в HttpOnly Cookie).
* **API-Ключи (ApiKeys):** Межсервисное взаимодействие и доступ для сторонних скриптов защищены через кастомный `ApiKeyMiddleware`.
* **Хэширование паролей:** Пароли пользователей защищены алгоритмом **BCrypt**.
* **Скрытые порты:** Бэкенд-сервисы и базы данных не имеют открытых наружу портов. Весь трафик идет через Nginx-прокси во фронтенд-контейнерах.

## Быстрый запуск

Для запуска приложения требуется установленный **Docker** и **Docker Compose**.

1. Клонируйте репозиторий.
2. В корневой папке проекта выполните команду:

```bash
docker-compose up -d --build
```

3. Дождитесь запуска всех контейнеров (около 1-2 минут).

### Доступ к приложениям

* **Игра (Пользовательский интерфейс):** [http://localhost:8080](http://localhost:8080)
* **Панель Администратора:** [http://localhost:8081](http://localhost:8081)
* **Панель RabbitMQ:** [http://localhost:15672](http://localhost:15672) *(Логин/Пароль: user / password)*

### Данные Администратора по умолчанию

При первом запуске автоматически создается учетная запись системного администратора.

* **Логин:** `admin`
* **Пароль:** `superadmin_password123`

*Вы можете войти под этими данными по адресу `http://localhost:8081`.*

## Тестирование

В проекте реализовано 13 интеграционных тестов с использованием `xUnit` и `Testcontainers` (автоматически поднимает временную БД и RabbitMQ для тестов).

Для запуска тестов необходим установленный **.NET 10 SDK** и **Docker** (для Testcontainers).

Выполните команду в корне проекта:

```bash
dotnet test
```

*Тесты покрывают регистрацию, авторизацию, покупку предметов, логику увеличения баланса с учетом RabbitMQ и проверку безопасности (API-ключи, JWT, блокировки).*

## Структура репозитория

* `/ClickService` — сервис отправки кликов в очередь.
* `/PlayerService` — сервис игроков, базы данных игроков и консьюмер RabbitMQ.
* `/StoreService` — сервис магазина и базы данных товаров.
* `/AdminService` — админские маршруты управления пользователями.
* `/SharedLibrary` — общая библиотека (Middleware, сообщения RabbitMQ, DTO).
* `/Frontend` — SPA Blazor-приложение для игроков.
* `/AdminFrontend` — SPA Blazor-приложение для администраторов.
* `/IntegrationTests` — проект с интеграционными тестами.
* `docker-compose.yml` — конфигурация развертывания.
* `Report.md` — академический отчет по проекту (для MS Word).
